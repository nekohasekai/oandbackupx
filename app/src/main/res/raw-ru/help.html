Режимы резервного копирования: Приложение может создать резервную копию только с APK, только данными или с тем и другим. Не установленное в настоящий момент приложение может быть восстановлено, только если существует копия APK.

Delete backup: Deletes the backup files for the chosen app.

Uninstall: Somewhat more aggressive than a normal uninstall. In addition to doing a normal uninstall via android commands, uninstalling from OAndBackupX deletes files the app might have left over in /data/app-lib/.

Enable / disable: Uses the android script `pm` to enable or disable an app. Disabling an app removes it from the normal user interface without uninstalling. This can be used for enabling or disabling an app for muliple users at a time (if the device has multiple users enabled). Users are identified with an id. 0 is the first user (owner).

Многопользовательская среда: Использование приложения в многопользовательской среде всё ещё является экспериментальным, но должно работать. Включение и отключение работает только после первоначальной установки (не обязательно из OAndBackupX) или восстановления приложения.

System-less приложения: Резервирование и восстановление system-less приложений, установленных в виде Magisk-модулей, не поддерживается. Попытка сделать это может привести к повреждению системы, вследствие чего придётся переформировать и переустановить её.

Навигация: В навигационной панели есть "Настройки", "Домой", "Массовая резервная копия", "Массовое восстановление" и "Резервные копии по расписанию" (в таком порядке). Количество приложений обновляется после их последней резервной копии, которая будет представлена в виде уведомления над кнопкой "Домой".

Приложения, использующие Хранилище ключей: Android имеет зашифрованное хранилище для конфиденциальных данных. Приложения, которые используют его для хранения учётных данных или каких-либо ключей, не могут быть успешно восстановлены. Два примера таких приложений — это Signal и Element(Riot).
