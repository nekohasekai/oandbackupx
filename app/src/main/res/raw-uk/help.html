Режими резервного копіювання: З додатка можна створити резервну копію лише її apk, лише її дані або обидва. Програму, не встановлену на даний момент, можна відновити лише за наявності файлу .apk. Три додаткові типи даних підтримуються додатково до основних даних: захищені пристроєм дані, зовнішні дані та дані obb.

Видалити резервну копію: Видаляє файли резервної копії для певного екземпляра резервної копії. "Видалити резервні копії" видалить усі резервні копії вибраного додатка.

Видалення: Дещо агресивніший, ніж звичайне видалення. На додаток до звичайного видалення за допомогою команд Android, видалення з OAndBackupX видаляє файли, які програма могла залишити в /data/app-lib/.

Enable / disable: Uses the android script `pm` to enable or disable an app. Disabling an app removes it from the normal user interface without uninstalling. This can be used for enabling or disabling an app for muliple users at a time (if the device has multiple users enabled). Users are identified with an id. 0 is the first user (owner).

Multiple users: Multi-user is still somewhat experimental but should work. When restoring in a multi-user scenario, `pm install -r $apk` gets called and subsequently the app is disabled for every user who has the app listed in /data/system/user/$user/package-restrictions.xml (unless the app is listed as enabled="1").
This can create problems for users installing the same app at some later point, but is necessary to prevent the app from being installed to all users at the same time. The context menu has an option to enable or disable apps which can be used if other users become unable to use a specific app due to disabling on restore.
Enabling and disabling only works after an initial install (not necessarily from OAndBackupx) or restore of the app.

Несистемні програми . Резервне копіювання та відновлення несистемних програм, встановлених як модулі Magisk, не підтримується, і ця спроба може призвести до збою системи, форматування та перепрошивання заново.

Navigation: The navigation bar has "Settings", "Home", "Batch backup", "Batch restore" and "Schedule backups" (in that order). The number of apps upgraded after their last backup will be presented as a notification on "Home" button.

Apps using Keystore: Android has an encrypted storage for sensitive data. Apps that make use of this to store login data or some keys for its function can't be restored successfully. Two examples of such apps are Signal and Element(Riot).
